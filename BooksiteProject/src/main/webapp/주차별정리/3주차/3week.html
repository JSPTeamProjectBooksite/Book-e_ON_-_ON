[09-26 월 개발작업]


안녕하세요 조장 이현우입니다.
한 주간 잘 쉬셨나요?

저번주에 DB를 만들고, 더미데이터를 집어넣고 sql문을 프로젝트 파일 내에 구상한 후 깃허브로 공유, DB 프로그램에 링크해서 공유, 수정 가능하게 만들어 두었습니다. 오늘은 기존에 직접적으로 값을 직어넣어두었던 부분을 지우고, DB에서 값을 가져오는 방식으로 고치겠습니다.

[09-28 수 개발작업]


어제 조원 한 분(이름은 말하지 않겠습니다. 하영씨)이 커밋을 하다가 몇몇 파일에 이상한 코드가 추가되는 일이 생겼는데요.

여러분이 코딩을 하다가 코드에 이상이 생기면, 즉시 하던작업을 멈추고 커밋을 하지 말아야합니다. 이미 푸쉬를 했다고 해도, 절대 이이상으로 커밋을해서 피해를 늘리시면 안됩니다. 우선 본인 로컬에서 작업 하던 파일만, 따로 옮겨 저장한 후 이전버전으로 돌려서 다시 커밋을 하는 식으로 다른 프로젝트까지 피해가 생기는 일은 막으셔야합니다. 지금 복구작업을 한 후 이번주는 계속 DB를 통한 페이지 기능 추가 작업를 합니다.

별도의 공지가 적은 점 역시 이렇다 할만한 작업이 이루어 지지 않고 있기 때문인데요. 제가 생각하기에는 저희조는 지금 진도가 느린편이라고 생각됩니다. 좀더 분발해서 페이지를 완성해 나가도록 합시다.


[09-29 목 개발작업]


몇 주째 같은 페이지 만 보고있습니다...

슬슬 이 코너 역시, 질려가는데요. 현제 DB를 이용해서 페이지 개발과 모델2(MVC패턴)에 대한 이해도 상승, 전반적인 페이지 디자인과 기능에 대해 포괄적으로 프로젝트를 성장 시키고 있는 중이라고 생각합니다. 3주차가 끝나니 슬슬 마무리하고 다음주에는 여태한 스킬을 활용하여 다른페이지를 늘리는 작업을 해볼려합니다. 각자 맡은 페이지에서 프로젝트와 자바에 대한 공부는 잘 이루어지고 있다고 생각합니다. 비록 프로젝트 결과가 더디더라도 해매지 말고, 잘 나아가셨으면 좋겠습니다.

과정이 한달도 남지 않자, 취직을 고려하는 조원이 늘고 있습니다. 저역시 흔들릴때가 있습니다.
"학원에선 프로젝트와 공부에 집중하고, 지금처럼 스킬을 쌓을 수 있는 시간이 없다"라고 생각했습니다만, 취직하는 학생이 한,두명 생기고 나서부턴 저역시 불안 합니다.

저나 다른조원이 아닌, 자기 자신을 믿고 프로젝트를 열심히 진행한 끝엔 좋은결과가 있을거라 생각하며 나아갑시다.

이번주는 DB를 이용하기 시작해서 DTO, DAO를 구상하고, controller로 페이지를 연결하는 작업을 했습니다.

1주차 기획, 2주차 구상과 레이아웃에 이어 3주차를 했는데 3주차는 마무리가 어설프다는 느낌이 듭니다.
저만 이런건 아니겠지요? 어찌됐던 아직은 JSP에서 배운 기술과 스택을 확실히 활용하고있지 못하다는 느낌입니다.

다음주(4주차)에는 여태했던 메인-상품-결제로 이어지는 페이지가 아닌, 관리자 페이지를 만들어볼 예정입니다.
관리자계정으로 로그인해서 DB값을 조회, 수정, 삭제하는 기능을 만들고, 책DB를 추가하기 편하게 하는 기능들도 만들어볼 예정입니다.
